# Discovery notes

This directory stores durable outputs from product-shaping sessions.

## Purpose

These notes bridge the gap between raw ideas and GitHub Issues.

They are intentionally lighter than ADRs and more durable than chat history.

## Lifecycle

1. Start with a discovery session note.
2. Refine the story map, slices, and open questions.
3. Extract follow-up `Spike` or `Task` issues.
4. Leave the note in place as historical reasoning, even after implementation
   begins.

## File naming

Use `YYYY-MM-DD-short-topic.md`.

## Template

Copy the structure from `SESSION-TEMPLATE.md`.
